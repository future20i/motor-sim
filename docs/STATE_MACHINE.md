**适用范围:** 泓慧能源 500kW/3kWh 飞轮UPS全链路仿真平台

---

## 1. 工作模式概述

仿真系统可在一定条件期间自动供应交流电力给关键负载。允许系统供电的工作模式有多种。
多数工作模式有多个出现其中的状态。

系统连续监控自身和市电电力。如有需要，无需操作员介入，即可在适用的模式之间自动切换。
检测和切换逻辑可确保工作模式改变是自动的且对于关键负载透明。

### 1.1 顶层工作模式

| 模式 | 符号 | 说明 | 负载受保护? |
|:--|:--|:--|:--|
| **Online** | `ONLINE` | 正常运行，飞轮待机 | ✅ 是 |
| **Discharge** | `DISCHARGE` | 飞轮向负载供电 | ✅ 是 |
| **AVR** | `AVR` | 自动电压调整（飞轮不可用） | ❌ 否 |
| **Bypass** | `BYPASS` | 市电直通旁路 | ❌ 否 |
| **Shutdown** | `SHUTDOWN` | 完全断电 | ❌ 否 |
| **Precharge** | `PRECHARGE` | 直流母线预充电 | ❌ 否 |
| **Calibration** | `CALIBRATION` | 参数辨识/校准 | — |
| **Manual** | `MANUAL` | 维修诊断模式 | — |

## 3. ONLINE 模式子状态

| 状态 | 转速/条件 | 说明 |
|:--|:--|:--|
| **MOTOR_DRIVE** | 0 → 4400 RPM | 市电驱动飞轮加速，尚未达到放电能力 |
| **CHARGING** | ≥4400 RPM | 飞轮有足够能量支持负载，但仍未达待机速度 |
| **STANDBY** | 8800 RPM | 全速待机，从输入获取约 200W 维持损耗 |

### 3.1 启动配置 (Auto Load and Start)

| 设置 | 模式 | 行为 |
|:--|:--|:--|
| **0** | 手动加载/手动启动 | 上电后保持 SHUTDOWN，需手动操作 |
| **1 (默认)** | 自动加载/手动启动 | 上电自动进入 BYPASS，需手动按键使 ONLINE |
| **2** | 自动加载/自动启动 | 上电自动 BYPASS → AVR → ONLINE，无需操作 |

## 4. DISCHARGE 模式子状态

| 状态 | 说明 |
|:--|:--|
| **DISCHARGING** | 飞轮向负载供电，输入接触器打开，静态开关 OFF |
| **PHASE_MATCH** | 市电恢复，输出与市电同步锁定 |
| **WALK_IN** | 负载从飞轮转移回市电（先接后断） |

### 4.1 放电触发条件

| 类型 | 检测机制 | 阈值 (默认) | 用户可调? |
|:--|:--|:--|:--|
| **RMS电压高** | RMS电压超出上限 | >110% Vnom | ✅ |
| **RMS电压低** | RMS电压低于下限 | <90% Vnom | ✅ |
| **快速电压瞬变** | 瞬时电压超出范围 | ±15% Vnom | ✅ |
| **频率高** | 频率超出上限 | >50.5 Hz | ✅ |
| **频率低** | 频率低于下限 | <49.5 Hz | ✅ |
| **直流母线高** | 母线超出上限 | >850V | ✅ |
| **直流母线低** | 母线低于下限 | <650V | ✅ |

## 5. BYPASS 模式子状态

| 状态 | 说明 |
|:--|:--|
| **OFFLINE** | 系统因故障锁定在旁路，电力流经内部旁路电路 |
| **VERIFY_SIGNALS** | 验证遥测正确性（启动/故障恢复时使用） |
| **AUTO_START** | 默认启动状态，转入 AVR → ONLINE |

---

## 6. AVR (自动电压调整) 模式

AVR 是特殊的工作模式，输出电压被调整到标称设定值但负载不受保护。

| 进入条件 | 说明 |
|:--|:--|
| **系统启动** | BYPASS → AVR → MOTOR_DRIVE → ONLINE |
| **飞轮错误** | 没有足够的飞轮支持负载，降级为 AVR |
| **飞轮预热** | 冷启动时飞轮需要预热到 >30°C |

### 6.1 AVR 子状态

| 状态 | 条件 | 说明 |
|:--|:--|:--|
| **AUTO_START** | 初始进入 | 等待飞轮获得 45 RPM |
| **CHARGING** | ≥45 RPM | 飞轮充电中，达到标称转速→ONLINE |
| **VERIFY_SIGNALS** | 飞轮错误恢复 | 验证遥测正确性后→CHARGING |

## 7. SHUTDOWN 模式

| 进入条件 | 说明 |
|:--|:--|
| **EPO (紧急断电)** | 按下系统柜正面 EPO 按钮 |
| **REPO (远程紧急断电)** | 远程 REPO 按钮激活 |
| **EMO (紧急模块关闭)** | 单个 MMU 紧急关闭 |
| **内部故障** | 引起系统关机的严重故障 |

### 7.1 关机后复原

```
EPO/REPO → SHUTDOWN → 查明并修正原因
                     → 复原 EPO 断路器 (Off→On)
                     → 钥匙开关 Online→Reset→Online (保持1秒)
                     → 执行启动程序
```

---

## 8. CALIBRATION 模式

| 状态 | 说明 |
|:--|:--|
| **CALIBRATE** | 校准遥测通道 (工厂初始校准) |
| **POWER** | RPS (转子位置传感器) 校准期间为飞轮供电 |
| **AUTO_TUNE** | 调整飞轮整流以最大化充电效率 |

## 9. 状态指示器 (绿/黄/红)

| 颜色 | 含义 | 保护负载 | 示例 |
|:--|:--|:--|:--|
| **🟢 绿色** | 正常工作 | ✅ 是 | ONLINE-STANDBY, NORMAL |
| **🟡 黄色** | 注意 (Notice) | ⚠️ 不一定 | 风扇故障, 高温警告, 真空度超限 |
| **🔴 红色** | 警报 (Alarm) | ❌ 否 | EPO, 过载, 多重风扇故障, 飞轮锁定 |

**Notice vs Alarm**:
- **Notice (注意)**: 非关键的系统错误，不会触发模式切换，需要关注但可继续运行
- **Alarm (警报)**: 关键的系统错误，需要操作员立即采取措施，可能触发模式切换

## 10. 代码状态枚举

```python
class SystemMode(Enum):
    """系统工作模式"""
    ONLINE = "online"           # 正常运行
    DISCHARGE = "discharge"     # 飞轮供电
    AVR = "avr"                 # 自动电压调整
    BYPASS = "bypass"           # 市电直通
    SHUTDOWN = "shutdown"       # 完全断电
    PRECHARGE = "precharge"     # 母线预充
    CALIBRATION = "calibration" # 校准
    MANUAL = "manual"           # 维修模式

class OnlineSubState(Enum):
    """Online 模式子状态"""
    MOTOR_DRIVE = "motor_drive"   # 飞轮加速中
    CHARGING = "charging"         # 已充电可放电
    STANDBY = "standby"           # 全速待机

class DischargeSubState(Enum):
    """Discharge 模式子状态"""
    DISCHARGING = "discharging"   # 飞轮供电中
    PHASE_MATCH = "phase_match"   # 相位同步
    WALK_IN = "walk_in"           # 负载转移回市电

class StatusLevel(Enum):
    """状态级别"""
    NORMAL = "normal"     # 🟢 正常
    NOTICE = "notice"     # 🟡 注意
    ALARM = "alarm"       # 🔴 警报
```
